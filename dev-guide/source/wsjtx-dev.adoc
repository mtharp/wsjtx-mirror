=== Linux Makefile Example

In order to build _WSJT-X_ form source, install the following packages:

[source,shell]
----------
sudo apt-get install gcc g++ gfortran make libfftw3-dev \
libpulse-dev libhamlib-dev qtbase5-dev  pulseaudio subversion \
libgfortran3:i386 libc6-i386 libqt5multimedia5-plugins \
qtmultimedia5-dev
----------

adjust install method to you distribution package manager.

==== WSJT-X Build and Run Dependencies
[horizontal]
+*gcc*+:: GNU C Compiler
+*g{plus}{plus}*+:: GNU C{plus}{plus} Compiler
+*gfortran*+:: GNU Fortran or F90
+*make*+:: GNU Make
+*libqt5multimedia5-plugins*+:: QT5 Media Plugins

=== Windows CMake Example

The critical portion of building _WSJT-X_ on Windows is setting up the
environment and defining paths. Once complete, subsequent builds can be
accomplished in two steps:
* Open the +*wsjt-env terminal*+
* Build _WSJT-X with wsjtx-build-cmake.bat

[source,shell]
---------
+*wsjtx-build-cmake.bat -r*+
---------

IMPORTANT: With the exception of {svn}, The entire development environment is
contained within: +*C:\wsjt-env*+. Using the +*wsjtx-env terminal*+, previous
installs of +*QT*+ or +*CMake*+ should should not affect the build.

==== Directory Setup
* Open Windows CMD Terminal
* Use the following commands to create the required folder structure

[caption="Create Directory Structure:"]
.C:\wsjt-env\.. .. ..
----------
REM -- One command per line, Copy and Paste

c:
mkdir wsjt-env\src wsjt-env\downloads wsjt-env\fftw3
mkdir wsjt-env\wsjtx-build\Debug wsjt-env\wsjtx-build\Release
mkdir wsjt-env\wsjtx-install\Debug wsjt-env\wsjtx-install\Release

REM -- Close the CMD window
----------

==== Download Development Packages

* Save the following packages to +*C:\wsjt-env\downloads*+

[horizontal]
+*Qt5 SDK*+:: {qt5_framework} - Qt 5.2.1 Windows-32, (MinGW 4.8, OpenGL)
+*CMake*+:: {cmake} - Win32 Installer
+*Hamlib*+:: {hamlib} - Hamlib 1.2.15.3.zip
+*FFTW3*+:: {fftw3} - 32-bit version: fftw-3.3.3-dll32.zip
+*SVN*+:: {svn} - SilkSVN 1.8.5.x 32-bit

==== QT5.2.1 Install
* Run the Installer
* Change Install Path to: +*C:\wsjt-env\Qt5*+
* Check: Associate common files types with Qt Creator
* Select Components: Use Default Selections
* License Agreement, read and select
* Start Menu: Use default selections, select next, then install
* Go grab a coffee, this takes about +*5*+ minutes
* At Finish, launch Qt Creator to verify it works, then exit.

==== CMake Install
* Run the Installer
* Agree to Terms
* Select: Do [red]*#not#* add to system PATH
* Change Install Path to: +*C:\wsjt-env\CMake*+

==== SilkSVN Install
* Run the Installer
* Agree to Terms
* Change Install Path to: +*C:\wsjt-env\SlikSvn*+
* Select Typical, then Install

==== Extract Hamlib
* Unzip folder hamlib-1.2.15.3.zip
* Rename folder +*hamlib-win32-1.2.15.3 => hamlib*+
* Move folder +*hamlib  =>  C:\wsjt-env\hamlib*+

==== Extract FFTW3
* [red]*#Caution#*: FFTW3 does *not* have a top level folder
* Move +*fftw-3.3.3-dll32.zip  =>  C:\wsjt-env\fftw3*+
* Then extract in +*C:\wsjt-env\fftw3*+
